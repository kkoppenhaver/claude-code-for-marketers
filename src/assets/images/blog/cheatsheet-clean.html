Hooks, MCP Servers & Skills

Claude Code — Quick Reference
Use… Hooks to enforce rules & automate at lifecycle events
Use… MCP Servers to connect Claude to external tools & data
Use… Skills to package expert workflows into reusable playbooks
Hooks
What it is

User-defined shell commands, LLM prompts, or agent spawns that execute automatically at specific lifecycle points.

Example config
settings.json 
{
  "hooks": {
    "PostToolUse": [{
      "type": "command",
      "matcher": "Edit|Write",
      "command": "prettier --write \"$CC_EDIT_FILE_PATH\""
    }]
  }
}
Key details
3 types: command (shell), prompt (single-turn LLM), agent (multi-turn sub-agent)
Events: PreToolUse PostToolUse Stop SessionStart and more
Scoping: User, Project, or Local settings.json — project deny rules can’t be overridden locally
Security: Execute with full user permissions — no sandbox
MCP Servers
What it is

External servers that expose tools, resources, and prompts to Claude Code via the Model Context Protocol.

Example config
.claude/settings.json 
{
  "mcpServers": {
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y",
        "@modelcontextprotocol/server-github"]
    }
  }
}
Key details
3 primitives: Tools (actions), Resources (data), Prompts (templates)
Transport: stdio (local subprocess) or Streamable HTTP (network endpoint)
Scoping: Project-scoped (committed) or user-global — deploy once, everyone connects
CLI shortcut: claude mcp add <name> to configure interactively
Skills
What it is

On-demand playbooks packaged as markdown files that Claude loads when invoked. Like downloading Kung Fu in The Matrix.

Example config
SKILL.md 
---
name: code-review
description: Structured code review.
  Use when the user asks to
  review a PR or diff.
---

# Instructions
1. Read the current git diff
2. Analyze for issues
3. Output structured feedback
Key details
Structure: SKILL.md (playbook), /references (templates), /scripts (executable code)
Frontmatter: YAML name + description tells Claude when to invoke it
Sharing: Commit to .claude/skills/ in version control — team-wide access
vs CLAUDE.md: CLAUDE.md is always loaded; skills are invoked on-demand when needed
