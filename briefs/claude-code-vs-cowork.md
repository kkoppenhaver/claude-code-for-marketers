## Outline

### 1. The quick answer

* Same AI under the hood (same Claude models, same Agent SDK)

* Code = terminal-first, runs on your machine with full system access

* Cowork = GUI tab in the Claude Desktop app, runs in a sandboxed VM

* Neither replaces the other — they're designed for different workflows

### 2. What Claude Code actually is

* Brief intro for readers who might not know (this is a broad-capture post)

* Terminal tool, but you're typing plain English, not code

* Originally built for developers, but non-technical people use it for all kinds of work

* Link to existing intro posts (installing Claude Code, don't be scared of the terminal)

### 3. What Claude Cowork actually is

* Launched January 2026, lives inside the Claude Desktop app

* Describe a folder of files, tell it what you want, it works

* Runs in a sandboxed VM — safer for people who don't want to think about system access

* Great at producing polished documents: Excel with real formulas, PowerPoint, Word docs, PDFs

* Newer features: Projects, Dispatch (assign tasks from your phone), Computer Use

### 4. The real differences (comparison table + narrative)

* **Interface:** Terminal vs. GUI — accessibility tradeoff

* **What they produce:** Code produces anything your computer can do; Cowork optimized for business documents

* **Where they run:** Direct system access vs. sandboxed VM

* **Extensibility:** Code extends through skills, hooks, and configuration; Cowork extends through plugins and connectors

* **Learning curve:** Cowork is easier day one; Code is more powerful once you're comfortable

* Include a comparison table (similar to deep research doc but simplified, marketing-focused)

### 5. Misconceptions that need to die

* "Claude Code is only for developers" — I use it to write this blog, manage Google Docs, research competitors

* "You need Cowork for non-coding tasks" — Code handles non-coding work just fine

* "Only Cowork can use your browser" — Code can too (Playwright MCP, browser automation)

* "They have different intelligence levels" — same models, different interfaces

* "Cowork is dumbed-down Code" — it's optimized differently, not simplified

### 6. Where each one shines (with marketing examples)

* **Cowork wins:** Quick document creation (polished slide deck for a client meeting), organizing a messy folder of campaign assets, tasks where you don't want to think about setup

* **Code wins:** Recurring workflows (weekly reporting, content repurposing), anything involving your codebase (landing pages, website changes), deep integrations via MCP (Google Docs, WordPress, email platforms), complex multi-step automations

* **Keanan's browser automation story:** Used Cowork for a 1-hour browser automation where no API was available — it handled that well. But for API-connected workflows, Code is faster and more reliable.

### 7. So which should you use?

* If you're brand new to AI agents and want immediate results: start with Cowork

* If you're willing to invest a few hours learning: Claude Code will take you further

* If you already use Claude Code: Cowork is a nice complement for quick document tasks, not a replacement

* You don't have to choose — they're on the same subscription
